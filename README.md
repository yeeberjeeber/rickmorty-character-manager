# The Story Behind the Rick & Morty Character Manager

Imagine that the **Rick and Morty multiverse** has become a complete administrative nightmare.<br/>
<br/>
After years of portal jumping, cloning experiments, and alternate timelines, the Rick Sanchez Central Finite Curve database is overflowing with thousands of characters from different universes.<br/>
<br/>
Every time Morty Smith visits a new dimension, they encounter:<br/>
- new versions of themselves<br/>
- alternate Ricks<br/>
- strange alien species<br/>
- characters who may or may not already exist in the system<br/>
<br/>

Soon the **Citadel of Ricks** realizes something: they **can't keep track of everyone anymore**. <br/>
<br/>

So they hired a junior developer to build a **Character Management System**.<br/>
<br/>
The mission is to create a React application that allows Citadel to:<br/>

- **View characters** across the multiverse<br/>
- **Search and filter** them efficiently<br/>
- **Inspect detailed information** about each character<br/>
- **Organize them into manageable data structures**<br/>
<br/>

The system pulls character data from the **Rick and Morty API**, which acts as the **interdimensional database**.
